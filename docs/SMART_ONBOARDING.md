# Smart Contextual Onboarding & Context Management

This document describes the Smart Contextual Onboarding system and first-class context management features.

## Overview

The Smart Onboarding system automatically bootstraps organizational context during signup and provides full user control to review, approve, edit, and delete all inferred context.

## Architecture

### Backend Components

#### Database Schema (`supabase/init/003_context_management.sql`)

- **context_objects**: Stores all business context with type, content, source, confidence, and status
- **context_evidence**: Stores detailed evidence for context objects
- **context_links**: Graph relationships between context objects
- **onboarding_sessions**: Tracks user onboarding progress

#### Models (`backend/models/context.py`)

Pydantic models for:
- `ContextObject`: Full context object with metadata
- `OnboardingSession`: Onboarding session tracking
- `CompanyInfo`: Company information input

#### Services (`backend/services/context_db.py`)

`ContextDatabaseService`: CRUD operations for context objects and onboarding sessions

#### Reasoning Pipelines (`backend/reasoning/context_extraction.py`)

- `CompanyContextExtractor`: Extract company profile from website and basic info
- `IndustryRegulationMapper`: Map industry to applicable regulations
- `CompetitiveLandscapeExtractor`: Identify competitors and market adjacencies
- `PersonaGenerator`: Generate constraint-based customer personas
- `OnboardingPipeline`: Orchestrates all extraction pipelines

#### API Endpoints (`backend/main.py`)

**Onboarding:**
- `POST /onboarding/start`: Start onboarding and trigger context discovery
- `GET /onboarding/{session_id}/context`: Get discovered context for review

**Context Management:**
- `GET /context`: List context objects with filters
- `GET /context/{id}`: Get specific context object
- `PATCH /context/{id}`: Update context object
- `DELETE /context/{id}`: Soft delete context object
- `POST /context/{id}/confirm`: Confirm context object
- `POST /context/{id}/reject`: Reject context object

### Frontend Components

#### Onboarding Flow (`frontend/src/pages/onboarding/`)

1. **CompanyInfoStep**: Collect company name, website, industry
2. **ContextDiscovery**: Show real-time progress of context extraction
3. **ContextReview**: Review, edit, approve, or reject discovered contexts
4. **OnboardingFlow**: Orchestrates the complete flow

#### Context Management (`frontend/src/pages/context/`)

- **ContextManagement**: Post-onboarding CRUD interface for all context objects
  - Filters by type, status, source
  - Edit, confirm, reject, delete actions
  - Evidence inspection

#### API Client (`frontend/src/lib/contextApi.ts`)

Functions for all context and onboarding API calls.

## Context Object Types

- **company**: Company profile (description, business model, geography, size)
- **industry**: Industry classification
- **regulation**: Applicable regulatory frameworks
- **competitor**: Competitive landscape
- **persona**: Customer personas (constraint-based, not fictional)
- **assumption**: Baseline business assumptions

## Context Object Lifecycle

1. **Created** (status: `pending`, source: `inferred`)
   - Generated by reasoning pipelines during onboarding
   - Has confidence score and evidence references

2. **User Review**
   - User can view, edit, confirm, or reject
   - Edits increment version number

3. **Final State**
   - `confirmed`: User approved, treated as active knowledge
   - `rejected`: User rejected, not used in reasoning
   - `deleted`: Soft deleted (deleted_at set)

## Security & Trust Principles

- **Nothing inferred is auto-confirmed**: All AI-generated context requires user review
- **All data sources are logged**: Evidence references tracked
- **Clear UI distinction**: Inferred vs confirmed vs user-created
- **No legal claims**: System makes no compliance or legal determinations
- **User control**: All context is editable and deletable

## Usage

### Starting Onboarding

```typescript
const session = await startOnboarding({
  name: "Acme Corp",
  website: "https://acme.com",
  industry: "Technology"
});
```

### Reviewing Context

```typescript
const response = await getOnboardingContext(sessionId);
// response.contexts contains all discovered context objects
// response.progress shows discovery progress
```

### Managing Context

```typescript
// List contexts
const contexts = await listContexts({ status: 'pending', type: 'regulation' });

// Confirm a context
await confirmContext(contextId);

// Edit a context
await updateContext(contextId, { content: updatedContent });

// Delete a context
await deleteContext(contextId);
```
